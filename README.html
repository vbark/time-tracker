Time Tracker

A native macOS app for tracking work hours, calculating overtime balance, and keeping your time data synced via iCloud Drive.

SwiftUI macOS 14+ Swift 6 Menu Bar App iCloud Sync

Features

⏱

Timer

Start/stop with live elapsed display. Survives app restarts — the original start time is preserved in UserDefaults.

✏️

Manual Entry

Add time entries with start/end times, optional note, and off-day toggle. Entries land on the selected calendar date.

📋

Entry Management

Double-click to edit, right-click to delete. Full table view with time range, duration, note, and type columns.

📅

Calendar

Month view with navigation. Days are color-coded: work, off, today, selected.

📊

Statistics

Selected day, week, month, and overall stats. Hours worked, expected, balance, and days — with a hero balance display.

🖥

Menu Bar

Always-visible menu bar icon with popover: timer status, today's progress bar, quick start/stop, and overall balance.

☁️

iCloud Drive

CSV stored in iCloud Drive for automatic sync. Local backup kept as fallback. Configurable in Settings.

🌗

Dark & Light Mode

Follows macOS system appearance natively. Custom accent colors adapt to both modes automatically.

CSV Format

Fully backward-compatible with the original Python time-tracker app. You can drop in an existing time_log.csv and it reads it directly.

ColumnFormatDescription
dateYYYY-MM-DDEntry date
start_timeHH:MMWork start (24h)
end_timeHH:MMWork end (24h)
durationHH:MMCalculated duration (00:00 for off days)
is_off_daytrue/falseOff days don't affect balance
is_overtime_takentrue/falseReserved field for future use
noteFree textOptional note for the entry

Example:

date,start_time,end_time,duration,is_off_day,is_overtime_taken,note
2026-05-14,08:15,12:30,04:15,false,false,Morning session
2026-05-14,13:00,17:00,04:00,false,false,Afternoon session
2026-05-15,09:00,17:00,00:00,true,false,Public holiday

Storage

File Locations

LocationPathPurpose
Primary (iCloud) ~/Library/Mobile Documents/com~apple~CloudDocs/TimeTracker/time_log.csv Auto-syncs via iCloud Drive
Backup (local) ~/Library/Application Support/TimeTracker/time_log_backup.csv Offline fallback & crash recovery
Timer state UserDefaults Running timer survives app restarts
Settings UserDefaults Daily target hours, storage path, preferences
Dual-storage pattern: Every save writes to both primary and backup. On load, the app picks whichever file was most recently modified. If the primary is unavailable, the backup is used automatically with a warning indicator.

Custom Storage Path

Open Settings (⌘,) to point the primary file at any location — a different iCloud folder, a Dropbox directory, or a local path. The backup always stays in Application Support.

Timer Persistence

Unlike the Python version, the timer survives app quits, crashes, and device restarts:

  1. On start: The original start time is saved to UserDefaults immediately.
  2. On restart: The app detects the saved start time and resumes counting from it — no time is lost.
  3. On stop: A time entry is created with the original start and current end time. UserDefaults is cleared.

Statistics

All statistics are pre-computed whenever entries change. The balance logic works identically to the Python app:

  • Daily balance = hours worked − daily target (default 8h)
  • Off days are fully excluded — they contribute 0h, not −8h
  • Week uses ISO weeks (Monday → Sunday)
  • Expected hours shown as "tracked days × target / weekdays × target"

Architecture

TimeTrackerApp (@main) ├── WindowGroup → MainView │ ├── Left Column │ │ ├── TimerView (start/stop, elapsed display) │ │ ├── QuickActionsBar (today, refresh, export) │ │ ├── ManualEntryView (time inputs, note, off-day) │ │ ├── DaySummaryView (total, balance, count, status) │ │ └── EntriesListView (Table with edit/delete) │ └── Right Column │ ├── StatisticsView (hero + 4 sections) │ └── CalendarCardView (month grid, navigation) ├── MenuBarExtra (.window) → MenuBarView │ ├── Timer status + controls │ ├── Today's progress bar │ └── Quit button └── Settings → StorageSettingsView ├── Daily target hours └── Storage path management

Key Patterns

  • @Observable ViewModel — Single source of truth with @MainActor isolation. All derived state (day totals, weekly balance, calendar colors) is pre-computed on mutation, keeping view bodies trivial.
  • CSVService — Stateless read/write. Handles legacy files missing the "note" column. Respects quoted fields with commas.
  • StorageService — Coordinates dual-file writes, picks newest on load, reports status/warnings.
  • TimerPersistenceService — UserDefaults-backed timer state, checked on launch for recovery.

Build & Run

Quick Start (app bundle)

cd TimeTracker
./build-app.sh
open build/TimeTracker.app

Install to Applications

cp -R build/TimeTracker.app /Applications/

Development (live rebuild)

swift build
swift run

With Xcode (optional)

xcodegen generate
open TimeTracker.xcodeproj

Requirements

  • macOS 14.0+
  • Swift 6.0+ (included with Xcode 16+ or Command Line Tools)
  • Xcode 16+ (optional — only needed for .xcodeproj)
